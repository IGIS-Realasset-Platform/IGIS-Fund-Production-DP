# Project Rules

- 작업을 완료하여 정상적으로 빌드 및 검증이 완료된 경우, 반드시 자동으로 로컬 git commit까지만 수행하며, git push는 절대 실행하지 않습니다.
- 통합 업무 보드(PmoTaskBoardStaging.jsx 및 관련 테이블 컴포넌트)의 데이터 열 너비(width, min-width, max-w 등 테이블 컬럼 폭 간격)는 절대 임의로 수정하거나 변경하지 않으며, 사용자가 직접 수정한 폭 간격을 온전히 보존합니다.
- 사용자가 명시적으로 지시하지 않은 UI/UX 디자인 변경, 레이아웃 수정(예: 그리드 배열, 여백 조절, 선/테두리 추가 등)은 절대 임의로 수행하지 않습니다. 오직 기능 구현 및 명시된 스타일 수정 지시에만 집중합니다.
- **[Capacitor / 모바일 테스트 주의사항]** 로컬 Xcode (iOS) 테스트를 위해 Live Reload 설정을 하거나 `capacitor.config.json`을 건드릴 때, 개발 서버 포트는 반드시 **8082** (`staging` 모드 포트)를 사용하십시오. 5173이나 8081 등 기본 포트를 임의로 찾거나 백그라운드에서 새 서버를 띄우지 마십시오. 사용자는 이미 `8082` 포트로 서버를 구동 중이므로, 설정 시 `http://<로컬IP>:8082` 형태로만 맞추고 `cap sync ios`를 실행하면 됩니다.
