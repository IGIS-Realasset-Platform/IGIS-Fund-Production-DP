# Logistics Leasing Permission Matrix

사용자 리스트를 받기 전까지 실제 사용자 매핑은 하지 않습니다. 이 표는 권한별 UI 노출 구조의 기준입니다.

| 기능 | 읽기 | 쓰기 | 수정 | 삭제 | 종합 관리 |
| --- | --- | --- | --- | --- | --- |
| 탭 조회 | 허용 | 허용 | 허용 | 허용 | 허용 |
| KPI/표/지도 상세 drawer 조회 | 허용 | 허용 | 허용 | 허용 | 허용 |
| 필터/검색/저장 view 로컬 조작 | 허용 | 허용 | 허용 | 허용 | 허용 |
| 메모/이슈 신규 등록 | 숨김 | 허용 | 허용 | 허용 | 허용 |
| 기존 행 수정 | 숨김 | 숨김 | 허용 | 허용 | 허용 |
| 삭제/비활성화 | 숨김 | 숨김 | 숨김 | 허용 | 허용 |
| OpenDART 동기화 | 숨김 | 숨김 | 숨김 | 숨김 | 허용 |
| 건축물대장 동기화 | 숨김 | 숨김 | 숨김 | 숨김 | 허용 |
| snapshot refresh/cache clear | 숨김 | 숨김 | 숨김 | 숨김 | 허용 |
| 사용자/권한 관리 | 숨김 | 숨김 | 숨김 | 숨김 | 허용 |

검증 기준:
- 권한 없는 버튼은 disabled가 아니라 DOM 노출 최소화가 원칙입니다.
- 실행성 기능은 Supabase Edge Functions를 통해서만 호출합니다.
- 공개 fallback 화면에서는 모든 실행성 기능을 read-only 상태로 표시합니다.
