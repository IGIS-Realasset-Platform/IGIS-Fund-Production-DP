이식 방향

main 웹 플랫폼에 `/mobile` 또는 `/m` route를 추가하고, 기존 PC 플랫폼 route는 그대로 유지합니다.
이 브랜치를 그대로 merge하지 않습니다. Flutter 앱의 화면 구성과 Supabase 로직만 모바일 웹 컴포넌트로 재작성합니다.
Android 앱과 iPhone 웹의 탭명, 카드 정보, 작성 흐름을 맞춰 하나의 모바일 제품처럼 보이게 합니다.

알림 우선 설계

OS와 무관하게 안정적으로 맞출 기준은 "웹페이지가 열려 있는 동안"의 in-page realtime 알림입니다. 브라우저 notification은 권한과 브라우저 정책 차이가 있으므로 보조 수단으로 둡니다.

1. 업무 이벤트글, 댓글, task 저장
2. DB Trigger수신자별 iota_notifications row 생성
3. RealtimePC/모바일 웹이 내 user_id row를 구독
4. UI 알림toast, badge, 알림 목록 즉시 갱신
5. 보조 알림허용 시 browser notification, 진동, 소리

비주얼라이즈 규칙

task는 `task_name`을 노란 포인트 제목으로, `next_action`을 보조 본문으로 표시합니다.
log는 title, 본문 3줄, workspace/project/triage/visibility/status/writer/comment chip으로 압축합니다.
권한 없는 log는 본문 대신 lock 상태를 보여주고 detail 진입을 막습니다.
작성은 모바일 bottom sheet로 띄우고 `Task 등록`과 `협업 글작성`을 segmented control로 전환합니다.

작업자에게 넘길 체크포인트

`/mobile` route에서 기존 mobile blocker를 우회하고, 기존 Supabase AuthContext를 재사용해야 합니다. 실제 DB 쓰기 전에는 task table별 column 차이와 `metadata.permissions.individuals`의 email/name 기준을 한번 더 맞추는 것이 좋습니다.
