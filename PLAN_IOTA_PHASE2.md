# 이오타 CFT 플랫폼 Phase 2 시스템 기획안: 실전 운영화 및 사용성 혁신

본 문서는 이오타(IOTA) CFT 협업 플랫폼의 실전 운영 및 비즈니스 통제 고도화를 위한 Phase 2 시스템 기획서임. 

Phase 1에서 수파베이스(Supabase) 데이터베이스 기틀 마련과 기초 데이터 마이그레이션이 완료됨에 따라, Phase 2는 **'엑셀 원장의 1:1 기계적 재현'에서 탈피하여 '실무 사용성(Usability) 기반의 데이터 가치 창출'**을 최우선 목표로 정의함. 한찬호 대리 작성 엑셀 원장의 내재적 금융/개발 연계 규칙을 추출하여, 플랫폼 사용자 관점에서 최적화된 UX 워크플로우로 녹여내는 데 중점을 둠.

---

## 1. Phase 2 핵심 기획 방향 및 설계 철학

*   **기계적 그리드 렌더링 배격:** 대량의 셀 데이터를 브라우저에 단순 열거하는 스프레드시트 덤프 방식을 지양함. 사용자의 직무 맥락(PM, LFC, 임원)에 필요한 정보만 시기적절하게 노출하는 사용자 중심 인터페이스로 개편함.
*   **엑셀 계산 규칙의 시스템 내재화 (Rule Engine):** 대주단 금리 스택, 기금 인출(Drawdown) 시점 연계, 우선순위 계산식 등 수작업 엑셀 수식을 데이터베이스 트리거와 UI 내 가상 시뮬레이터로 자동화함.
*   **의사결정 맥락(Context) 중심의 아카이빙:** 단순 텍스트 기록을 넘어, 리파이낸싱 경과나 의사결정 로그가 자산 타임라인과 연계되어 생성 즉시 자동 보호/통제되는 신뢰성 있는 협업 아키텍처를 구현함.

---

## 2. 사용성 기반 5대 핵심 기능 명세

### 1) 스프레드시트 뷰 기반 스마트 인라인 에디터 (Inline Edit UX)
*   **기존 방식:** 세부 정보 수정을 위해 무거운 모달 창을 띄우거나 엑셀 전체를 새로 업로드해야 하는 불편이 존재함.
*   **개선 기획:**
    *   각 부서별 태스크 그리드 내에서 상태, 마감일, 담당자, 다음 액션 등의 셀을 **더블클릭하는 즉시 인라인 입력 폼으로 변환**하여 수정 가능하도록 구현함.
    *   병목 여부(Blocker) 및 의사결정 필요 여부는 마우스 **원클릭 토글**로 즉각 백엔드 Supabase DB에 반영 및 임원 대시보드에 실시간 전파되도록 함.
    *   화면 포커싱 유지 및 비동기(Async) 저장 기법을 적용하여 엑셀 수준의 고속 편집 사용성을 확보함.

### 2) 다차원 비주얼 캐피탈 스택 & NOI 시뮬레이터 (Visual Capital Stack UX)
*   **기존 방식:** 브릿지론에서 본 PF, 리파이낸싱으로 이어지는 자본 구조와 금융 기관별 조달 금리 추이를 엑셀 시트 행으로 대조해야 하여 리스크 파악이 지연됨.
*   **개선 기획:**
    *   수파베이스의 에쿼티(Equity) 및 론(Loan) 상세 내역을 연계하여, 자본 구조 변화 추이를 **누적 막대 차트(Stacked Bar Chart)** 형태로 자동 시각화함.
    *   금리 인상 또는 대주단 구성 변경 시, 예상 연간 순영업소득(NOI)과 자본환원율(Cap Rate)의 임계치 한계를 시뮬레이션할 수 있는 **인터랙티브 슬라이더(Interactive Slider) 뷰**를 구축하여 복잡한 수식 연산을 UI 내부로 감춤.

### 3) 맥락 인지형 인물 자동완성 연동 (Autocomplete Stakeholder Catalog)
*   **기존 방식:** 외부 파트너(시행사, 시공사, 대주단 등)의 연락처를 찾기 위해 별도의 엑셀 주소록 시트를 열고 수작업으로 복사해 붙여넣어야 했음.
*   **개선 기획:**
    *   업무 코멘트 및 회의록 입력 창(`LogWriteBox`, `MeetingWriteBox`)에서 **이름 초성 또는 단어 타이핑 시 `iota_stakeholder_master` 테이블을 백그라운드로 실시간 조회**하여 매핑하는 자동완성(Autocomplete) 엔진을 구축함.
    *   입력 완료 시 이메일, 전화번호, 소속 정보가 담긴 **비주얼 태그(Chip)** 형태로 본문에 삽입되어 파트너 정보를 즉시 조회할 수 있는 연계 사용성을 제공함.

### 4) 자동 락킹(Auto-Locking) 기반 의사결정 맥락 타임라인
*   **기존 방식:** 이력 관리 시 누락이 발생하거나, 결정 번복 과정에서 과거 로그가 오염될 수 있는 보안 리스크가 존재함.
*   **개선 기획:**
    *   PF 조건 변경 등 전사 차원의 결정 사항을 기록하는 `iota_decision_logs`에 로그가 인서트되는 즉시, 시스템 레벨에서 **`is_locked = true`**로 전환하여 무단 수정을 자동 방어함.
    *   해당 락은 어드민 권한(`iota_seoul_pilot_members.role_code = 'master'`)을 가진 임원진만 해제할 수 있도록 Supabase RLS(Row Level Security) 단에서 강력히 바인딩함.
    *   축적된 로그는 연도/분기별 인터랙티브 카드 타임라인 위젯으로 렌더링되어 프로젝트 역사를 손쉽게 탐색할 수 있도록 함.

### 5) 회의체 상정 자동 필터링 대시보드 (A-Grade Escalation)
*   **기존 방식:** 매주 주간 업무 회의 진행 시, 엑셀 필터링 기능을 수작업으로 조작하여 보고 대상을 추려내야 함.
*   **개선 기획:**
    *   업무 우선순위 가중치가 80점 이상이거나 Blocker(병목)가 활성화된 중요 태스크를 시스템이 자동 감지하여 **'A등급 즉시 상정' 카드로 최상단에 자동 배치**함.
    *   회의 참석 임직원은 필터 조작 없이 대시보드 진입만으로 병목 현황이 심각한 안건을 즉시 확인하고 조율할 수 있음.

## 3. 모바일 활용도 극대화 및 하이브리드 알림 체계

*   **반응형 '핫 테스크' 피드 제공 (Mobile-First):** 
    *   모바일 환경에서 거대한 엑셀 테이블 렌더링을 차단하고, 대신 최근 2주 동안 업데이트되었거나 Blocker가 켜진 업무들만 모아둔 **카드 피드(Hot Tasks Feed) 형태로 간소화 노출**함.
    *   이동 중에도 한 손으로 터치하여 이슈 상황 및 담당 다음 액션을 조회할 수 있도록 스크롤 중심의 모바일 레이아웃을 구현함.
*   **이벤트 기반 스마트 푸시 알림:** 
    *   업무 원장에 Blocker 표시가 켜지거나 실무자가 액션 아이템 배정자로 태깅되는 즉시, 연동된 이메일 또는 슬랙(Slack) 브릿지를 통해 지연 리스크와 담당자 정보를 실시간으로 대상 부서원 및 임원진에게 전송함.

---

## 4. 데이터베이스 및 인프라 보안 강화 (RLS 수립)

*   **테이블별 RLS(행 레벨 보안) 전면 가동:** 
    *   Phase 1 검증 과정에서 도출된 `funds` 등 7개 무방비(UNRESTRICTED) 테이블에 대해 보안 패치(`enable_rls_unrestricted_tables.sql`)를 전면 적용하여 외부 악성 API 요청으로부터 플랫폼 자산 데이터를 보호함.
    *   사용자 소속 세션 정보(`auth.jwt()`)와 `iota_seoul_pilot_members` 권한 매핑 정보를 대조하여, 소속 부서가 일치하거나 임원 등급인 경우에만 UPDATE/DELETE 권한이 유효하게 열리도록 DB 단에서 격리 제어함.
